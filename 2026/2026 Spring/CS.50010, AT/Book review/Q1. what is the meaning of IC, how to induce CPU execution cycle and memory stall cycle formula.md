# 1️⃣ IC (Instruction Count)란 무엇인가

**IC = Instruction Count**

즉

```
프로그램을 실행하는 동안 실행되는 총 instruction 수
```

예를 들어 프로그램이

```
add
load
branch
store
...
```

같은 명령을 **10억 개 실행했다면**

```
IC = 10^9
```

입니다.

CPU 실행 시간 공식의 가장 기본 형태는

```
CPU execution time
= IC × CPI × Clock cycle time
```

입니다.

---

# 2️⃣ Clock Cycle이 무엇인가

CPU는 **clock signal**로 동작합니다.

예를 들어 CPU가

```
3 GHz
```

이면

```
1초에 3 × 10^9 번 clock이 뜸
```

입니다.

clock 하나의 길이는

```
Clock cycle time
= 1 / clock frequency
```

예

```
3 GHz → clock cycle = 0.333 ns
```

즉

```
clock cycle = CPU의 가장 기본 시간 단위
```

입니다.

---

# 3️⃣ CPI (Cycles Per Instruction)

CPI는

```
instruction 하나를 실행하는데 필요한 clock cycle 수
```

입니다.

예

|instruction|cycle|
|---|---|
|add|1|
|load|2|
|branch|1|

평균이

```
CPI = 1.2
```

이 될 수 있습니다.

---

# 4️⃣ CPU execution time 공식

그래서 기본 공식은

```
CPU execution time
= IC × CPI × Clock cycle time
```

입니다.

하지만 실제 CPU는 **memory stall**이 있습니다.

그래서 책에서는 다음처럼 나눕니다.

```
CPU execution time
= (CPU clock cycles + Memory stall cycles) × Clock cycle time
```

즉

```
총 cycle = CPU 계산 cycle + memory 기다리는 cycle
```

입니다.

---

# 5️⃣ 왜 Memory stall cycles 공식이 저렇게 되는가

책의 공식은 다음입니다.

```
Memory stall cycles
= IC × (Memory accesses / Instruction)
        × Miss rate
        × Miss penalty
```

각 항의 의미를 보면 이해됩니다.

---

## (1) IC

총 instruction 수

```
IC
```

---

## (2) Memory accesses / Instruction

instruction 하나당 memory 접근 수입니다.

예

|instruction|memory access|
|---|---|
|add|0|
|branch|0|
|load|1|
|store|1|

---

이 문제에서는

```
loads + stores = 50%
```

라고 했습니다.

즉

```
50% instruction → memory access
```

입니다.

---

하지만 중요한 점이 하나 있습니다.

### instruction fetch도 memory access입니다.

CPU는 instruction을 실행하려면

```
instruction memory
```

에서 instruction을 가져와야 합니다.

즉

```
모든 instruction → 1번 memory access
```

입니다.

---

그래서 총 memory access는

```
instruction fetch = 1
data access = 0.5
```

입니다.

따라서

```
Memory accesses / Instruction
= 1 + 0.5
= 1.5
```

입니다.

---

그래서

```
0.5가 아니라 1.5
```

가 되는 것입니다.

---

# 6️⃣ Miss rate

문제에서

```
miss rate = 2%
```

입니다.

즉

```
memory access 100번 중 2번 miss
```

입니다.

---

# 7️⃣ Miss penalty

문제에서

```
miss penalty = 25 cycles
```

입니다.

즉

cache miss가 나면

```
memory에서 가져오는데 25 cycle
```

걸립니다.

---

# 8️⃣ 이제 전체 공식 이해

Memory stall cycles

```
= IC
× (memory access / instruction)
× miss rate
× miss penalty
```

값을 넣으면

```
= IC × 1.5 × 0.02 × 25
```

계산하면

```
1.5 × 0.02 = 0.03
0.03 × 25 = 0.75
```

따라서

```
Memory stall cycles = IC × 0.75
```

입니다.

---

# 9️⃣ CPU execution time 계산

cache miss가 있는 경우

```
CPU cycles = IC × CPI
```

문제에서

```
CPI = 1
```

입니다.

따라서

```
CPU cycles = IC × 1
```

입니다.

---

전체 cycle

```
CPU cycles + stall cycles
```

즉

```
= IC × 1 + IC × 0.75
= 1.75 IC
```

그래서

```
CPU execution time
= 1.75 IC × Clock cycle
```

입니다.

---

# 🔟 cache miss가 없으면

miss가 없다면

```
stall cycles = 0
```

입니다.

그래서

```
CPU execution time
= IC × 1 × Clock cycle
```

입니다.

---

# 11️⃣ speedup 계산

speedup은

```
execution time_old
-------------------
execution time_new
```

입니다.

여기서는

```
1.75 IC / 1 IC
```

입니다.

그래서

```
= 1.75
```

입니다.

즉

```
cache miss가 없으면 1.75배 빠름
```

입니다.
