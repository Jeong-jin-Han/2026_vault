# 질문1: https://shields.io/badges 사용방법
네, 물론입니다 😊  
지금 쓰신 예시는 **shields.io 배지를 HTML `<img>` 태그로 직접 삽입**한 형태인데, 구조만 이해하시면 **원하시는 문구·색·로고로 자유자재로 만들 수 있습니다**.  
아래에서 **구조 → 만드는 순서 → 커스터마이징 → 당신 케이스 예시**까지 차근차근 설명드릴게요.

---

## 1️⃣ 지금 쓰신 코드, 하나씩 뜯어보기

```html
<img src="https://img.shields.io/badge/Research_Interests-181717?style=for-the-badge&logo=openai&logoColor=white"/>
```

이 한 줄은 사실 **URL 파라미터 조합**입니다.

### 🔹 기본 뼈대

```text
https://img.shields.io/badge/<LABEL>-<MESSAGE>?<OPTIONS>
```

당신 코드에 대응시키면:

| 부분                    | 의미             |
| --------------------- | -------------- |
| `Research_Interests`  | 왼쪽 텍스트 (LABEL) |
| `181717`              | 배지 색상 (HEX)    |
| `style=for-the-badge` | 큼직한 Hero 스타일   |
| `logo=openai`         | OpenAI 로고      |
| `logoColor=white`     | 로고 색           |

> ⚠️ `_` 는 공백으로 렌더링됩니다  
> 공백을 쓰고 싶으면 `_` 또는 `%20` 둘 다 OK입니다

```
https://img.shields.io/badge/Research_Interesets-181717?style=for-tho-badge&logo=openai&logoColor=white

```

---

## 2️⃣ 만드는 순서 (이 순서 그대로 하시면 됩니다)

### ✅ Step 1. 왼쪽 텍스트 정하기

```text
Research_Interests
Special_Purpose_Architecture
Math_Driven_Design
```

---

### ✅ Step 2. 색 정하기

|예시|의미|
|---|---|
|`181717`|GitHub 다크 블랙|
|`blue`|기본 파랑|
|`purple`|연구/수학 느낌|
|`critical`|빨강 (강조)|

```text
https://img.shields.io/badge/Research_Interests-181717
```

---

### ✅ Step 3. 스타일 선택 (중요 ⭐)

Hero 섹션에서는 **무조건 이걸 추천**드립니다:

```text
style=for-the-badge
```

---

### ✅ Step 4. 로고 붙이기 (선택)

```text
logo=openai
logo=nvidia
logo=github
logo=python
logo=pytorch
```

> shields.io는 **Simple Icons**를 사용합니다  
> 👉 [https://simpleicons.org](https://simpleicons.org/) 에서 정확한 이름 확인 가능

# 질문2: js, jsx, tsx 각각의 차이점에 대해서 설명해줘
아주 핵심적인 질문입니다.  
Next.js + React + TypeScript를 쓰고 계신 지금 타이밍에 **정확히 정리하고 가는 게 맞습니다.**

아래에서 **“언어 차이 → JSX 가능 여부 → Next.js에서의 실전 선택 기준”** 순서로 깔끔하게 설명드리겠습니다.

---

## 1️⃣ 한 줄 요약부터

|확장자|의미|JSX 사용|타입 체크|요즘 권장도|
|---|---|---|---|---|
|`.js`|순수 JavaScript|❌|❌|❌ (레거시/스크립트용)|
|`.jsx`|JS + JSX|✅|❌|⚠️ (JS 프로젝트 한정)|
|`.ts`|TypeScript|❌|✅|⚠️ (로직/유틸 전용)|
|`.tsx`|TS + JSX|✅|✅|✅ **표준**|

👉 **Next.js + React + TS → 거의 전부 `.tsx`**

---

## 2️⃣ 각각 정확히 뭐가 다른가?

### 🔹 `.js` (JavaScript)

- 가장 기본
    
- 타입 없음
    
- JSX 문법 ❌
    

```js
function add(a, b) {
  return a + b
}
```

📌 **React 컴포넌트용으로는 거의 안 씁니다.**  
(Next.js에서도 특별한 이유 없으면 피합니다)

---

### 🔹 `.jsx` (JavaScript + JSX)

- JavaScript + React 문법
    
- 타입 체크 ❌
    
- JSX 가능 ✅
    

```jsx
export default function Button() {
  return <button>Click</button>
}
```

📌 TypeScript를 안 쓰는 React 프로젝트에서는 흔함  
📌 **TS 프로젝트에서는 굳이 쓸 이유 없음**

---

### 🔹 `.ts` (TypeScript)

- JavaScript + 타입
    
- JSX ❌
    
- 로직, 설정, 유틸 함수에 최적
    

```ts
export function add(a: number, b: number): number {
  return a + b
}
```

📌 **React 컴포넌트는 못 씀**  
📌 대신:

- `lib/*`
    
- `utils/*`
    
- `types.ts`
    
- `config.ts`
    

이런 곳에 최적

---

### 🔹 `.tsx` (TypeScript + JSX)

- TypeScript + JSX
    
- 타입 체크 ✅
    
- React 컴포넌트 전용
    

```tsx
type Props = {
  label: string
}

export default function Button({ label }: Props) {
  return <button>{label}</button>
}
```

📌 **Next.js + React + TS의 사실상 표준**  
📌 지금 프로젝트에서 가장 많이 써야 하는 확장자

---

## 3️⃣ Next.js 기준 “언제 뭘 쓰나” (실전 기준)

이게 제일 중요합니다.

### ✅ `.tsx`를 쓰는 곳

- `app/page.tsx`
    
- `app/layout.tsx`
    
- `components/*.tsx`
    
- UI 컴포넌트 전부
    

👉 JSX가 있으면 **무조건 `.tsx`**

---

### ✅ `.ts`를 쓰는 곳

- `lib/github.ts`
    
- `utils/*.ts`
    
- `types/*.ts`
    
- 설정 파일 (`next.config.ts`, `tailwind.config.ts`)
    

👉 JSX가 없으면 **`.ts`**

---

### ❌ `.jsx` / `.js`는 언제?

- 이미 JS-only 프로젝트일 때
    
- 빌드 스크립트 (`scripts/*.js`)
    
- Node 실행 스크립트
    

📌 **Next.js + TS 환경에서는 거의 안 씁니다**

---

## 4️⃣ 흔히 생기는 오해들 (중요)

### ❌ “.tsx는 무거워서 필요한 곳만 써야 한다?”

→ ❌ 완전히 오해입니다.  
빌드 결과는 동일하고, **개발자 경험만 좋아집니다.**

### ❌ “className 많은 파일은 tsx가 부담스럽다?”

→ ❌ 오히려 타입 덕분에 안정성이 올라갑니다.

---

## 5️⃣ 추천 파일 전략 (딱 이렇게 하세요)

```txt
components/
  Button.tsx
  Card.tsx
  ProjectCard.tsx

lib/
  github.ts
  utils.ts

app/
  page.tsx
  projects/page.tsx
  layout.tsx
```

- **JSX 있음 → 무조건 `.tsx`**
    
- **JSX 없음 → `.ts`**
    
- `.js/.jsx`는 최소화
    

---

## 6️⃣ 지금 질문의 스샷(확장자 버튼들) 맥락에서 정리

그 버튼들이 `.js / .jsx / .tsx`를 보여주는 이유는:

- “이 문서를 어떤 언어 기준으로 볼래?” 선택용
    
- **실제 프로젝트 선택 가이드와는 다릅니다**
    

👉 실전에서는 고민할 필요 거의 없음  
👉 **Next.js + TS = `.tsx` & `.ts` 두 개만 기억**

---

## 🎯 한 문장으로 정리

> Next.js + React + TypeScript 환경에서는  
> **JSX가 있으면 `.tsx`**,  
> **JSX가 없으면 `.ts`**,  
> 나머지는 거의 쓰지 않습니다.

다음으로 자연스럽게 이어지는 질문은 보통:

- “그럼 `components/ui`랑 `components`는 어떻게 나누는 게 좋을까?”
    
- “types는 한 파일로 모을까, 컴포넌트 옆에 둘까?”
    

어느 쪽이 더 궁금하신가요?
